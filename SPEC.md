# SPEC.md — template_autopoiesis

> **Public exemplar** — DOI forthcoming

## Design Contract

`template_autopoiesis` is a deterministic project generator with a falsifiable honesty contract. Every phase is checkable.

---

## Phases

### P0 — Grammar definition ✅
- Grammar defined in `manuscript/config.yaml` under `autopoiesis:`
- `parse_grammar()` validates structure: non-empty slots, no duplicate options, known deps
- Grammar hash: 16-char SHA-256 prefix of canonical JSON

### P1 — Seeded expansion ✅
- `expand(grammar, seed)` uses `_digest_index` (SHA-256, no entropy)
- Spec includes schema_version, seed, grammar_hash, selections, deps, primitive_domain
- `spec_hash` = 16-char SHA-256 of canonical spec JSON

### P2 — Byte-stable materialize ✅
- `materialize(spec, out_root, template_root)` writes complete child project
- Primitive kernel sources vendored with import rewriting
- `provenance.json` records tree hash and file list
- Two calls with same spec produce identical tree hash

### P3 — Recompute verify ✅
- `verify_child(child_root)` reads provenance.json, re-reads all files, recomputes hash
- Never calls `assert` (returns CheckReport)
- Catches: tampered file, missing file, corrupt provenance

### P4 — Seal ✅
- `build_payload(spec_hash, tree_hash, seed)` → JSON
- `qr_matrix(data)` → 2D bool array (uses qrcode if available, fallback stub)
- `seal.json` is non-repudiation artifact

### P5 — Primitive library ✅
- 5 domains: optimization, dynamics, statistics, signal, graph
- 8 kernels total with analytic ground-truth expected outputs
- Negative controls: at least one per domain

### P6 — Mutation meta-gate ✅
- `test_meta_teeth.py` parametrized over KNOWN_DOMAINS
- Constant-success stub must fail per domain
- Real primitive must pass per domain

### P7 — Honesty manifest ✅
- `build_manifest(project_root)` inspects live AST
- `verify_honesty(project_root)` scans manuscript prose
- All structural evidence must pass

### P8 — Reserved slot honesty ✅
- `generate_variables` reports PRODUCT_SIZE, EFFECTIVE_PRODUCT_SIZE, RESERVED_SLOT_COUNT
- Abstract explicitly reports both nominal and effective sizes

### P9 — Property-based testing ✅
- Hypothesis: grammar product, expand determinism, double materialize, verify clean/tamper
- Stress: boundary seeds, 1000-seed, Merkle invariants

### P10 — Full manuscript render (partial ✅)
- Manuscript files: 00-06, 99, preamble.md, config.yaml
- Figures generated by `01_generate_manuscript_assets.py`
- Cover art by `generate_cover_art.py`
- PDF rendering via infrastructure pipeline (Chrome/Pandoc required, not gated in tests)

## Honest Scope Notes

- **Byte-stability**: within-platform only (same Python version, same OS)
- **Child PDF rendering**: not supported in standalone child context
- **Multi-file vendor deps**: not implemented (single-file modules only)
- **Domain claims**: none — this is a demonstration generator
