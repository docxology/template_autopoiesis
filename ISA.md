## Problem

The manuscript's page layout was loose (2.5cm margin, default body font) and had
no cover image on the title page despite a cover-art generator already existing
in `src/cover_art.py`. Deeper investigation (R8 generator-verification gate)
surfaced three latent defects behind that surface ask: the manuscript's
self-reported `Tests`/`Coverage` line was a hardcoded literal (371/99.94%)
completely disconnected from the real suite; `src/manuscript_figures.py` (71
statements) had zero test coverage; and the project's own Stage-02 analysis
pipeline silently failed on 3 of 9 scripts (a CLI entrypoint and a library
helper wrongly swept into analysis-script discovery, plus a numeric-prefix
ordering bug that ran the seal step before any child existed to seal).

**Iteration 2 (2026-07-06, same day):** after the first pass, the user opened
the actual rendered PDF and found it "a bit light weight" — correctly:
~1,100 words across 7 sections, zero embedded figures (the 3 real figures
generated by `01_generate_manuscript_assets.py` were never referenced in any
manuscript section — orphaned outputs), and a bibliography with exactly one
self-citation placeholder. The ask: substantially expand the writing, add
more figures, add real references, then re-render.

## Vision

Someone opening the rendered PDF sees a denser, professionally laid-out page
with the existing cover art actually on the title page — and every claim in
the manuscript, including the "Tests: N · Coverage: X%" line, is a number this
session's own generator produced, not a number someone typed once and forgot.
The analysis pipeline runs clean end-to-end with no swallowed failures.

## Out of Scope

- Restoring the cover art's QR seal / gradient glow / seed dots (tracked in

  simplification, not part of this session's ask).
- Actually publishing (Zenodo deposit, GitHub push, DOI minting) — a
  credentialed, irreversible action; assessed readiness only, left for
  explicit user trigger.
- Fixing `scripts/pipeline/stage_01_test.py --project templates/<name>`'s test

- Closing the `sealing.py`/`verify.py`/`cli.py` coverage gaps below 90% each —
  the aggregate floor is met; these are pre-existing, already-tracked TODO
  items.

## Principles

- A manuscript's self-reported metrics must be traceable to a generator that
  ran this session, never a literal (R8: inherited-premise gate).
- A fix has as many arms as the bug has call-sites — grep every copy before
  editing one (R14).
- Config-driven typography (geometry/fontsize forwarded to pandoc) beats
  hand-declared LaTeX packages that can silently clash with renderer-injected
  ones.

## Constraints

- No mocks — the coverage/test-count fix must run a real pytest+coverage
  subprocess, not synthesize a plausible number.
- Must match the repo's own canonical CI coverage methodology
  (`branch = true`) so the manuscript's self-reported percentage is
  comparable to the actual gate, not a different (looser) measurement.
- Thin-orchestrator pattern: computation in `src/`, I/O (subprocess
  invocation, file writes) in `scripts/`.
